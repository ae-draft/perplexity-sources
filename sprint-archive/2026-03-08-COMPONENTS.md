## 📝 Методология

**Источник данных:**
- Поле `Компоненты` из JIRA CSV выгрузки
- При отсутствии компонента: анализ названия и описания задачи

**Формулы:**
```
Точность (%) = (Факт / Оценка) × 100%
Коэффициент = Факт / Оценка
Рекомендация (%) = (Коэффициент - 1) × 100%
```

**Группировка:**
- `Discussions (Issues)` — отдельный компонент (НЕ смешивать с Misc)
- `Misc` + Win-версия = Client Defects (Win)
- Отчёты группируются по типу (Report Excel, Report Projects And Works, etc.)

**Статусы надёжности:**
| Статус | Спринтов | Надёжность |
|--------|----------|------------|
| 🔴 Критично | 1 | Низкая (только ориентир) |
| 🟡 1 спринт | 1-2 | Средняя (с осторожностью) |
| 🟢 Надёжно | 3-5 | Высокая (для планирования) |
| ✅ Статистически значимый | 6+ | Очень высокая |
