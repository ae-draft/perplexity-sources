## 📝 Методология

**Источник данных:**
- Поле `Компоненты` из JIRA CSV выгрузки
- При отсутствии компонента: анализ названия и описания задачи

**Формулы:**
```
Точность (%) = (Факт / Оценка) × 100%
Коэффициент = Факт / Оценка
Рекомендация (%) = (Коэффициент - 1) × 100%
Rolling Average = (Спринт 1 + Спринт 2) / 2
```

**Группировка:**
- `Discussions (Issues)` — отдельный компонент (дискуссии)
- `Client Defects (Win)` — проект "Дефекты Win-версии (3.x)" + ключ DEFECT-*
- `Misc` — задачи без указанного компонента

**Статусы надёжности:**
| Статус | Спринтов | Надёжность | Использование |
|--------|----------|------------|---------------|
| 🔴 Критично | 1 | Низкая (только ориентир) |
| 🟡 2 спринта | 2 | Средняя (с осторожностью) |
| 🟢 Надёжно | 3-5 | Высокая (для планирования) |
