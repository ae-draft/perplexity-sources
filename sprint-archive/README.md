# 📊 Архив спринтов

## Назначение

Эта папка предназначена для хранения отчётов по завершённым спринтам для накопления экспертизы и анализа точности оценок.

---

## Структура файлов

| Файл | Формат | Описание |
|------|--------|----------|
| `YYYY-MM-DD-COMMAND.md` | Markdown | Отчёт по завершённому спринту |
| `YYYY-MM-DD-COMMAND-raw.csv` | CSV | Выгрузка JIRA для спринта |

---

## Шаблон имени файла

```
[ГОД]-[МЕСЯЦ]-[ДЕНЬ]-[КОМАНДА].md
```

**Примеры:**
- `2026-02-22-FRONT.md` — Отчёт по команде FRONT за спринт 16.02-22.02
- `2026-02-22-BACK.md` — Отчёт по команде BACK за спринт 16.02-22.02
- `2026-03-02-FRONT.md` — Отчёт по команде FRONT за спринт 02.03-08.03

---

## Что включать в отчёт

1. **Сводка по спринту**
   - Запланировано задач
   - Завершено задач
   - Внеплановых задач
   - Невыполненных задач
   - Успешность спринта (%)

2. **Точность оценок**
   - Оценка (часы)
   - Факт (часы)
   - Точность (%)

3. **Факторы отклонений**
   - Причины перерасхода
   - Причины экономии
   - Внешние блокеры

4. **Рекомендации на следующий спринт**

---

## Как использовать для анализа

При построении нового отчёта по планированию спринта:

1. Открыть последние 3-5 отчётов из этого архива
2. Рассчитать среднюю точность оценок по команде
3. Применить коэффициент к новому планированию
4. Учесть повторяющиеся факторы рисков

---

## Пример расчёта коэффициента планирования

```
Команда FRONT:
- Спринт 16.02-22.02: 146% (перерасход 46%)
- Спринт 23.02-01.03: 132% (перерасход 32%)
- Спринт 02.03-08.03: 128% (перерасход 28%)

Средний коэффициент: 1.35
Рекомендация: Закладывать +35% к оценке при планировании
```

---

## Обновление

При завершении каждого спринта:
1. Построить отчёт анализа (использовать `analysis-pmt.md`)
2. Сохранить в эту папку с именем `YYYY-MM-DD-COMMAND.md`
3. При необходимости приложить raw-выгрузку JIRA
4. Обновить сводную таблицу в `reference.md` (Раздел 9)
