## 2. Форматы данных

### 2.1 Форматы дат

| Формат | Пример | Назначение |
|--------|--------|------------|
| ДД.ММ.ГГГГ | 22.02.2026 | Даты в отчётах |
| ДД.ММ-ДД.ММ | 16.02-22.02 | Периоды спринтов |
| ЧЧ:ММ | 14:30 | Время |
| ISO 8601 | 2026-02-22T20:02:42+0400 | Выгрузки JIRA |

---

### 2.2 Форматы времени

| Формат | Пример | Назначение |
|--------|--------|------------|
| Часы | 16.0 часов | Оценка задач |
| Дни + часы | 4 дня, 4 часа | Потраченное время |
| Секунды | 129600 секунд | Исходные данные JIRA |

**Конвертация:**
```
86400 сек = 1 день
3600 сек = 1 час
60 сек = 1 минута
```

## 3. Чек-листы

### 3.1 Чек-лист перед отправкой отчёта по спринтам

- [ ] Все команды определены по **НАЗВАНИЮ спринта**, не по отв. разработчику
- [ ] Все ФИО взяты **ТОЧНО из данных**, без додумывания отчеств
- [ ] Проверено наличие задач в статусе **"На паузе"**
- [ ] Все задачи без исполнителя помечены как **"Не назначен"**
- [ ] Количество задач в сводке **совпадает с фактическим**
- [ ] Нет задач в статусе **"Завершено", "Готово", "Слит", "Анализ"**
- [ ] Все ссылки на задачи корректны
- [ ] Проверены все спринты периода
- [ ] Проверены все статусы
- [ ] **Блокеры:** проверен спринт каждого блокера
- [ ] **Блокеры:** нет интерпретаций причин паузы без явных данных
- [ ] **Справочник:** все ФИО сверены с `reference.md`

---

### 3.2 Чек-лист для анализа рисков

- [ ] Все коэффициенты взяты из `reference.md`
- [ ] Все нормы нагрузки взяты из `reference.md`
- [ ] Расчёты проверены на корректность
- [ ] Выявлены все критические риски (Вес >= 3)
- [ ] Выявлены все переработки (> Максимум часов)
- [ ] Сформированы рекомендации по каждому риску

---

### 3.3 Чек-лист для анализа нагрузки

- [ ] Все сотрудники сопоставлены с ролями из `reference.md`
- [ ] Нормы часов взяты из `reference.md` (Раздел 4)
- [ ] Фактические часы посчитаны по всем задачам в спринте
- [ ] Выявлены переработки (> Максимум)
- [ ] Выявлены недогрузки (< 80% от нормы)
- [ ] Сформированы рекомендации по перераспределению

## 5. Уровни рисков (для таблиц)

## 6. Статусы блокеров (для таблиц)

| Статус | Значение | Иконка | Описание |
|--------|----------|--------|----------|
| Завершён | Блокер слит/завершён | ✅ | Задача готова к продолжению |
| Активен | Блокер в работе | ⏳ | Задача ожидает решения блокера |
| Внешний | Ожидание от клиента/внедрения | ⚠️ | Задача не может быть продолжена без внешних данных |
| Не запланирован | У блокера нет спринта | ❌ | Требуется приоритизация блокера |

## 8. Правила обновления шаблонов

| Ситуация | Действие |
|----------|----------|
| Изменён формат отчёта | Обновить соответствующий шаблон в Разделе 1 |
| Добавлен новый тип отчёта | Добавить новый шаблон в Раздел 1 |
| Изменены форматы данных | Обновить Раздел 2 |
| Добавлен новый чек-лист | Добавить в Раздел 3 |
| Найден новый кейс для примера | Добавить в Раздел 4 |

### Примечание о статусе "Анализ"

| Статус | Назначение | Включать в отчёт |
|--------|------------|------------------|
| Анализ | Разработчик изучает задачу, оценивает трудоемкость, проверяет полноту информации | ❌ Нет |

**Правило:** Задачи в статусе "Анализ" не включаются в расчёт загрузки спринта, так как ещё не имеют финальной оценки.

**При обновлении:**
1. Обновить версию файла (например, 1.0 → 1.1)
2. Обновить дату последнего обновления
3. Добавить запись в историю изменений (если ведётся)
