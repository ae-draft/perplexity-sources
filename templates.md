## 2. Форматы данных

### 2.1 Даты и время

| Формат | Пример | Назначение |
|--------|--------|------------|
| ДД.ММ.ГГГГ | 22.02.2026 | Даты в отчётах |
| ДД.ММ-ДД.ММ | 16.02-22.02 | Периоды спринтов |

| ISO 8601 | 2026-02-22T20:02:42+0400 | Выгрузки JIRA |

```
86400 секунд = 1 день
3600 секунд = 1 час
60 секунд = 1 минута
```

## 3. Чек-листы

### 3.1 Чек-лист перед отправкой отчёта по спринтам

- [ ] Все задачи сгруппированы по спринтам (по НАЗВАНИЮ спринта)
- [ ] Все ФИО взяты из `reference.md`
- [ ] Все ссылки на задачи корректны
- [ ] Блокеры проверены (спринт блокера, статус)
- [ ] Статусы задач соответствуют правилам (`sprints-pmt.md` Раздел 3)
- [ ] Нет задач в статусе "Завершено", "Готово", "Слит", "Анализ"
- [ ] Выгрузка сверена с количеством задач в отчёте
- [ ] Все комментарии в формате "Требуется уточнение у [Автор]"
- [ ] Шаблоны взяты из `templates.md`

### 3.2 Чек-лист для анализа рисков

- [ ] Все факторы риска взяты из `reference.md` (Раздел 3)
- [ ] Вес риска рассчитан корректно
- [ ] Уровень риска определён по порогу
- [ ] Все задачи на паузе > 7 дней помечены как критические
- [ ] Все блокеры без спринта помечены как высокие
- [ ] Все FeedbackPending > 5 дней помечены как высокие
- [ ] Рекомендации сформированы для каждого риска
- [ ] Сроки эскалации указаны

### 3.3 Чек-лист для анализа нагрузки

- [ ] Все нормы нагрузки взяты из `reference.md` (Раздел 4)
- [ ] Роли сотрудников определены по `reference.md` или `department.md`
- [ ] Загрузка рассчитана по формуле: (Факт / Норма) × 100%
- [ ] Переработки выявлены (> 100%)
- [ ] Недогрузки выявлены (< 80%)
- [ ] Рекомендации сформированы для каждого сотрудника
- [ ] Визуализация нагрузки добавлена

### 4.5 Точность оценок (улучшенный формат)

```markdown
| Задача | Оценка (ч) | Факт (ч) | Точность | Отклонение | Статус |
|--------|------------|----------|----------|------------|--------|
| FRONT-3144 | 8.0 | 20.0 | 250% | +150% | 🔴 Перерасход |
| FRONT-3189 | 8.0 | 8.0 | 100% | 0% | 🟢 Точно |
| BACK-2240 | 9.0 | 2.5 | 28% | -72% | 🟢 Экономия |
```

**Как читать:**
- **250% [+150%]** — делали в 2.5 раза дольше (перерасход 150%)
- **100% [0%]** — точно в оценку
- **28% [-72%]** — сделали в 3.6 раза быстрее (экономия 72%)
